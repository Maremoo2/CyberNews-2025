# Dashboard Data Paths

This document explains where to place JSON data files so the CyberNews dashboard picks them up correctly in both local development and the GitHub Pages production deployment.

---

## Static incident data (bundled at build time)

These files are imported directly by `src/App.jsx` via ES module imports and are bundled into the JavaScript bundle at build time.

| File | Description |
|------|-------------|
| `data/incidents-2025-enriched.json` | Enriched incidents for 2025 |
| `data/incidents-2026-enriched.json` | Enriched incidents for 2026 |
| `data/metadata.json` | Pipeline metadata (last updated, last enrichment) |
| `data/learning-log.json` | Analyst learning/validation log |

**Adding a new year:** Create `data/incidents-YYYY-enriched.json`, then add an entry to `YEAR_DATA_MAP` in `src/App.jsx`:

```js
const YEAR_DATA_MAP = {
  2025: incidents2025,
  2026: incidents2026,
  2027: incidents2027,  // add import at top of file too
};
```

The year selector buttons and the default year are computed automatically from `YEAR_DATA_MAP`.

---

## Live / runtime data (served from `public/`)

The **AI Insights** (`AIInsights` component) and **Weekly Analysis** (`WeeklyAnalysis` component) fetch JSON files at runtime using `fetch()`. These files are **not** bundled — they must exist under `public/data/` in the repository (and therefore under `dist/data/` after `npm run build`).

These panels are only shown when the selected year equals `new Date().getFullYear()` (the live year).

### Daily digest files

- **Path:** `public/data/daily/YYYY-MM-DD.json`
- **Example:** `public/data/daily/2026-03-07.json`
- **How loaded:** The component tries today's date first, then falls back up to 3 days earlier.
- **Generated by:** `npm run daily-digest`

Minimal schema:

```json
{
  "date": "2026-03-07",
  "generated_at": "2026-03-07T06:00:00Z",
  "digest": {
    "summary": "Short narrative summary...",
    "novel_clusters": [
      { "theme": "...", "item_count": 3, "summary": "...", "why_matters": "..." }
    ],
    "continuation_clusters": [
      { "theme": "...", "item_count": 2, "summary": "..." }
    ]
  }
}
```

### Weekly brief files

- **Path:** `public/data/briefs/week_YYYY-WW.json`
- **Example:** `public/data/briefs/week_2026-10.json` (ISO week 10 of 2026)
- **How loaded:** The component tries the current ISO week, then falls back up to 3 prior weeks.
- **Generated by:** `npm run executive-brief`

Minimal schema:

```json
{
  "week_start": "2026-03-02",
  "week_end": "2026-03-08",
  "generated_at": "2026-03-08T06:00:00Z",
  "brief": {
    "top_5_things": [
      { "headline": "...", "summary": "...", "impact_level": "HIGH" }
    ],
    "new_patterns": [
      { "pattern": "...", "significance": "...", "change_indicator": "+12%" }
    ],
    "top_risks": [
      { "risk": "...", "likelihood": "HIGH", "affected_sectors": ["Finance"] }
    ],
    "watch_signals": [
      { "metric": "...", "threshold": "...", "action": "..." }
    ]
  }
}
```

### Weekly analysis files

- **Path:** `public/data/analysis/week_YYYY-WW.json`
- **Generated by:** `npm run aggregate-weekly`

## URL conventions

The Vite build uses `base: '/CyberNews-2025/'` for GitHub Pages. All `fetch()` calls in the dashboard use `import.meta.env.BASE_URL` as the path prefix, which resolves to `/CyberNews-2025/` in production and `/` in development:

```js
const basePath = import.meta.env.BASE_URL || '/';
const url = `${basePath}data/daily/2026-03-07.json`;
// dev:   /data/daily/2026-03-07.json
// prod:  /CyberNews-2025/data/daily/2026-03-07.json
```

This means files placed under `public/data/` are accessible at the correct URL in both environments.

---

## Graceful empty states

If no daily/weekly files are found after trying the fallback dates, the components display a friendly empty-state message rather than throwing an error. No action is required from the developer — simply add the files when they are available.
